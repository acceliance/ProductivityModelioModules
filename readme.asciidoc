== Where these modules fit: a data governance strategy

These modules are building blocks of an end-to-end *data governance* approach based on Modelio. The idea is to maintain a single, governed UML data model ‚Äî the source of truth ‚Äî and to derive every downstream technical artefact from it, keeping documentation, database schemas, code and analytics engineering aligned.

To set up the complete environment (Modelio configuration, shared metamodel, governance profiles and the recommended workflow), use the companion repository:

== The modules

Each module reads the elements you have enabled in your UML model (classes, attributes, associations, stereotypes) and produces a specific deliverable.

=== JavaOrmGenerator

Generates *Java persistence and API code* from the UML model, in tandem with the Modelio Java Designer.

* *ORM / JPA generation* ‚Äî `@Entity`, primary keys (`@Id`, generated or business keys), `@Column` (length, uniqueness), relationships (`@OneToMany`, `@ManyToOne`, `@ManyToMany`, `@OneToOne`, managed/back references), Hibernate `@Comment`, and configurable inheritance strategies (single fused table or joined tables).
* *REST API generation* ‚Äî OpenAPI / Swagger annotations (`@Schema`, `@Validated`, `@NotNull`, `@Size`), `oneOf` / discriminator mappings for inheritance, and stereotype-driven vendor extensions.
* *JSON generation* ‚Äî Jackson annotations (`@JsonClassDescription`, `@JsonPropertyDescription`, `@JsonIdentityInfo`) for clean serialization.
* *Configuration dictionaries* ‚Äî export the ORM / API / JSON configuration to Excel for review and bulk editing.
* *Scoped generation* ‚Äî select the artefacts to generate by use case.

=== SqlGenerator

Generates *SQL DDL scripts* from the UML model for the relational database of your choice.

* *Tables and columns* with type mapping to the selected target database.
* *Keys and constraints* ‚Äî primary keys (including composite), foreign keys for `1:*`, `*:1` and `*:*` (automatic junction tables), `NOT NULL` and `UNIQUE` constraints.
* *Inheritance* ‚Äî foreign keys linking child tables to their parent (mother) table.
* *Enumerations* ‚Äî generation of database `enum` types.
* *Documentation* ‚Äî `COMMENT ON TABLE` / `COMMENT ON COLUMN` from the model descriptions.
* *Options* ‚Äî optional `snake_case` column naming; export the SQL configuration to Excel.

=== DbtGenerator

Generates a *dbt (data build tool)* project skeleton from the UML model, ready to drop into your analytics engineering workflow.

* *`schema.yml`* ‚Äî models, columns, data types and constraints (`primary_key`, `foreign_key`, `not_null`) plus `relationships` tests referencing other models via `ref()`.
* *One `.sql` model per table* ‚Äî staging select with typed casts and inline documentation.
* *Keys* ‚Äî business keys, inheritance keys and foreign keys derived from the model.
* *Options* ‚Äî target database type mapping and `snake_case` naming; export the configuration to Excel.

=== Neo4jGenerator

Generates *Neo4j Cypher scripts* from the UML model to provision a graph database and its metamodel.

* *Cypher generation* ‚Äî produces both a *SCHEMA* script (nodes, relationships, properties and constraints) and a *METAMODEL* script describing the graph structure.
* *Nodes and relationships* ‚Äî labels, properties and typed relationships derived from classes, attributes and associations.
* *Keys and typing* ‚Äî business keys and per-property Cypher type mapping via the `Neo4jElement`, `BusinessKey` and `CypherType` stereotypes.
* *Push scripts* ‚Äî copy `push_schema.py`, `push_metamodel.py`, `requirements.txt` and `neo4j.env.example` into the Cypher output folder to load the generated scripts into a running Neo4j instance.
* *Documentation* ‚Äî export the configuration to Excel.

=== DatavaultBuilderGenerator

Generates a *Data Vault* configuration for https://www.datavault-builder.com[Datavault Builder] directly from the UML model.

* *Data Vault objects* ‚Äî Hubs, Satellites and Links derived from classes, attributes and associations.
* *Dimensional satellites* ‚Äî optional generation of dimensional satellites alongside business satellites.
* *Advanced patterns* ‚Äî hub aliases for self-relations, "merge in mother" for inheritance, business-key handling.
* *Export formats* ‚Äî Datavault Builder export format V1 and V2, with optional ZIP packaging and target-database selection.
* *Round-trip* ‚Äî import a model from an existing Datavault Builder database and assist database migration.
* *Documentation* ‚Äî export the configuration to Excel.

=== ModelGeneratorAI

Generates a *UML data model from a natural-language description* using generative AI, so you can bootstrap a model from a business brief in seconds.

* *AI providers* ‚Äî OpenAI (ChatGPT) and Anthropic (Claude), with configurable model and API key.
* *GenAI Continuity* ‚Äî merges the AI output into an existing model context, adding only the new classes, attributes and relations so you can iterate safely.
* *Direct instantiation* ‚Äî the generated classes, attributes, relationships and enumerations are created straight into your Modelio model.
* *Audit trail* ‚Äî every prompt and response is recorded in an audit log for traceability.

== Installation

. Install https://www.modelio.org[Modelio] (5.x).
. In Modelio, open the module catalog / manager and add the `.jmdac` files from the link:modules[`modules/`] folder of this repository.
. Select the module(s) on your project.
. (Recommended) Set up the full governance environment from https://github.com/acceliance/ModelioForDataGovernance[ModelioForDataGovernance].

[NOTE]
====
The modules load and run without a licence; only the generation volume is capped (see <<Free to use, with a volume limit>>). Install your Acceliance licence to remove the limit.
====
